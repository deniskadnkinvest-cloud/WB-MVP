# SELLER STUDIO (PAN.X VTON) — Полное Описание Проекта и Промпт для Бизнес-Плана

## ЧАСТЬ 1: ОПИСАНИЕ ПРОЕКТА

### 1.1 Что это

**Seller Studio** — это веб-приложение (SaaS), которое позволяет селлерам (продавцам на маркетплейсах Wildberries, Ozon и др.) генерировать профессиональные фотографии товаров на моделях с помощью искусственного интеллекта.

Селлер загружает фотографию своей одежды (раскладку, фото на вешалке или flat lay), выбирает параметры модели, позу, фон — и нейросеть генерирует студийное фото, где модель одета в эту одежду. Технология называется **Virtual Try-On (VTON)**.

### 1.2 Проблема, которую решает

Профессиональная фотосессия для карточки товара на маркетплейсе стоит:
- Фотограф: 3,000–15,000₽ за сессию
- Модель: 5,000–20,000₽ за сессию
- Студия: 2,000–5,000₽ за час
- Стилист/визажист: 3,000–10,000₽
- Ретушь: 500–2,000₽ за фото
- **Итого: 15,000–50,000₽ за один товар (5-10 фото)**

Seller Studio позволяет сделать то же самое за **секунды** и за **копейки**.

### 1.3 Целевая аудитория

- **Мелкие селлеры** (1-50 товаров): не могут позволить себе фотосессии, используют стоковые фото или фото от поставщика
- **Средние селлеры** (50-500 товаров): делают фотосессии, но это дорого и долго
- **Крупные селлеры и агентства** (500+ товаров): нужен поток контента, быстрая смена сезонов
- **Бренды и производители**: хотят видеть свою одежду на моделях разных типажей
- **Дропшипперы**: работают с китайскими поставщиками, нужно быстро делать карточки

### 1.4 Рынок (Россия)

- Wildberries: ~500,000 активных продавцов (2025-2026)
- Ozon: ~300,000 активных продавцов
- Яндекс.Маркет, Lamoda, и другие: ~100,000
- **Итого TAM: ~900,000 продавцов одежды и аксессуаров в RU-сегменте**
- Из них категория "Одежда и обувь" — крупнейшая (~30-40% от всех товаров)
- Потенциальный SAM (доступный рынок): ~200,000–350,000 продавцов

## ЧАСТЬ 2: ФУНКЦИОНАЛЬНОСТЬ ПРОДУКТА

### 2.1 Основные функции (уже реализовано)

1. **Загрузка одежды** — drag-and-drop мультизагрузка до 9 вещей одновременно (футболка + брюки + аксессуары = всё надевается на одну модель)
2. **Кастинг-рум** — выбор модели из пресетов (европейская, азиатская, африканская, plus-size и др.), мужские и женские
3. **Детализация модели** — настройка волос, макияжа, татуировок, телосложения, возраста
4. **Кастомные промпты** — описание модели текстом с нуля ("рыжая девушка 25 лет с веснушками")
5. **Мои модели** — сохранение сгенерированных моделей (identity preservation) с загрузкой рефов (3 фото: фронт, 3/4 лево, 3/4 право)
6. **Редактирование моделей** — модификация сохранённых моделей (добавить тату, изменить волосы) с превью и сохранением как новой
7. **Позы** — пресеты поз + свободный текст
8. **Ракурсы камеры** — full body, medium shot, close-up, 3/4 и др.
9. **Фоны/Локации**:
   - Пресеты (белый циклорама, улица, парк, кафе и т.д.)
   - "Мои локации" — оцифровка реальных мест (загрузка 2-5 фото реальной локации, нейросеть воспроизводит её как фон)
   - Свободный текст ("крыша небоскрёба на закате")
   - Модификация локаций (добавить закат, неоновые огни и т.д.)
10. **Форматы изображения** — 1:1, 3:4, 4:3, 9:16, 16:9
11. **Фотосессия** — параллельная генерация 3 или 5 фото с разных ракурсов одним нажатием
12. **Итеративное редактирование** — после генерации можно дописать "сделать модель выше / убрать тени / добавить очки" и перегенерировать
13. **Lightbox** — просмотр в полном размере
14. **Скачивание** — каждое фото отдельно
15. **Авторизация** — Firebase Auth (Google Sign-In)
16. **Облачное хранение** — Firebase Storage + Firestore для моделей, локаций, настроек
17. **Adaptive Fashion** — поддержка инклюзивной моды (модели с инвалидностью, протезы и т.д.)

### 2.2 Техстек

- **Frontend:** React 19, Vite 8, Framer Motion
- **Backend:** Express.js (Node.js), проксирующий запросы к Google AI
- **AI Engine:** Google Gemini API — модель `gemini-3.1-flash-image-preview` (кодовое название "Nano Banana 🍌")
- **Auth:** Firebase Authentication (Google Sign-In)
- **Storage:** Firebase Storage (фото моделей, локаций)
- **Database:** Firebase Firestore (метаданные, промпты, настройки юзера)
- **Деплой:** Vercel (фронт + API routes)

### 2.3 Архитектура генерации

Запрос на генерацию содержит:
- Фото одежды (1-9 изображений, base64)
- Текстовый промпт (~500-1500 символов: описание модели, позы, фона, инструкции по одеванию)
- (Опционально) Рефы модели — фотографии лица для identity preservation (до 5 фото)
- (Опционально) Рефы локации — фотографии реального места для воспроизведения (до 5 фото)

Всё отправляется в Gemini API как multimodal запрос → модель возвращает сгенерированное изображение.

## ЧАСТЬ 3: ЭКОНОМИКА ПРОДУКТА

### 3.1 Себестоимость одной генерации

Источник: [Официальная страница pricing Google AI](https://ai.google.dev/gemini-api/docs/pricing), раздел `gemini-3.1-flash-image-preview`.

**Формула:** Image output = $60 за 1,000,000 токенов.

| Разрешение | Токенов | Цена за 1 фото | В рублях (курс ₽95/$) |
|---|---|---|---|
| 0.5K (512px) | 747 | $0.045 | ₽4.27 |
| **1K (1024px)** | **1,120** | **$0.067** | **₽6.40** |
| 2K (2048px) | 1,680 | $0.101 | ₽9.60 |
| 4K (4096px) | 2,520 | $0.151 | ₽14.35 |

Для маркетплейсов оптимальное разрешение — 1K (1024×1024). Дальше при необходимости апскейлим отдельно.

**Дополнительно к output:** input токены (промпт + фото одежды + рефы), но это ~$0.001–0.003 на запрос — пренебрежимо мало.

**Итого себестоимость 1 генерации ≈ $0.067 ≈ ₽6.40**

### 3.2 Себестоимость фотосессии

- 5 фото = ₽32.00

## ЧАСТЬ 6: СТРАТЕГИЯ ПРИВЛЕЧЕНИЯ И УДЕРЖАНИЯ

### Привлечение (Acquisition)

- Telegram-каналы для селлеров WB/Ozon (тысячи каналов, активная аудитория)
- YouTube / Reels / Shorts — демонстрация "было-стало" (вирусный формат)
- Партнёрства с обучающими курсами для селлеров
- Бесплатный триал (5-10 генераций без подписки)

### Удержание (Retention) — Геймификация и привыкание

Ключевая задача — **подсадить селлера на привычку** экспериментировать:
- "Попробуй другую модель — вдруг конверсия вырастет?"
- "Добавь фото на улице — разнообразие в карточке повышает CTR"
- "Сезон сменился — обнови фото на зимний фон за 30 секунд"

Механики:
- **Бесплатные генерации за приглашение друга** (реферальная программа)
- **A/B тест карточек** — "какое фото лучше зайдёт?" → мотивация делать больше вариантов
- **Ежемесячный отчёт** — "вы сэкономили ₽X на фотосессиях"
- **Галерея лучших работ** — социальное доказательство и вдохновение
- **Сезонные фоны** — автоматическое обновление пресетов (зима, весна, лето, Black Friday и т.д.)

## Контекст

Я разрабатываю SaaS-продукт для русскоязычного рынка маркетплейсов (Wildberries, Ozon, Яндекс.Маркет). Продукт называется **Seller Studio** — это веб-приложение, которое позволяет селлерам (продавцам одежды) генерировать профессиональные студийные фотографии одежды на моделях с помощью ИИ.

**Как работает:** селлер загружает фото своей одежды (раскладку/flat lay) → выбирает параметры модели (пол, типаж, поза, фон) → нейросеть генерирует реалистичное студийное фото, где модель одета в эту одежду (Virtual Try-On). Генерация занимает 30 секунд.

**Техническая основа:** Google Gemini API (модель gemini-3.1-flash-image-preview). Приложение на React + Express.js + Firebase.

### Себестоимость

- **1 генерация изображения (1024×1024px) = $0.067 = ~₽6.40** (по официальному прайсингу Google: $60 за 1M output tokens, 1 изображение = 1,120 tokens)
- Фиксированные расходы: ~₽5,000–10,000/мес (хостинг, Firebase, домен)
- Цена НЕ снижается при увеличении объёма (flat rate) на уровне Developer API
- Есть Enterprise-тариф Google с возможными скидками за объём (но нужен контракт)

### Предварительные тарифы (мой черновик)

| Тариф | Цена | Лимит генераций | Маржа |
|---|---|---|---|
| Старт | ₽500/мес | 25 | 68% |
| Про | ₽3,000/мес | 100 | 79% |
| Бизнес | ₽10,000/мес | Безлимит (~500 реально) | 68-81% |

### Ключевые функции

- Мультизагрузка одежды (до 9 вещей = полный outfit на одной модели)
- Кастинг-рум: выбор модели по полу, этничности, телосложению + кастомное описание
- Сохранение и повторное использование "своих моделей" (identity preservation)
- Оцифровка реальных локаций (загрузка фото реального шоурума/магазина для использования как фон)
- Фотосессия: генерация 3 или 5 фото с разных ракурсов одним нажатием
- Итеративное редактирование результата
- Адаптивная мода (инклюзивность)

### Рынок

- ~900,000 активных продавцов на WB + Ozon + Яндекс.Маркет
- Категория "Одежда" — крупнейшая (30-40% товаров)
- Реальная фотосессия стоит ₽15,000–50,000 за товар
- Прямых русскоязычных конкурентов с нативным VTON пока нет

## Задача

Составь детальный бизнес-план и финансовую модель для Seller Studio. Включи:
