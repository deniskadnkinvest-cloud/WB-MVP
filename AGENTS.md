# AGENTS.md — Инструкция для Codex / AI-агентов (VTON MVP)

Привет! Этот проект представляет собой интерактивную примерочную на базе AI-генерации изображений.

## Основная структура
- `src/` — React-фронтенд (интерфейс примерочной, загрузка фото).
- `server.js` — Express-сервер для обработки API запросов примерки.
- `server-autocatalog.js` — скрипт автоматической каталогизации товаров.
- `api/` — Serverless Functions (Vercel), основная бизнес-логика.
- `BUSINESS_BRIEF.md` — подробное описание бизнес-модели продукта.

## ⛔ ОБЯЗАТЕЛЬНЫЙ ПРОТОКОЛ ПЕРЕД ДЕПЛОЕМ

**НИКОГДА не делай `vercel --prod` или `git push` без выполнения этих шагов:**

### Шаг 1 — Lint (обязателен)
```bash
npm run lint
```
- Если есть **errors** (не warnings) — **СТОП**. Исправь ошибки до деплоя.
- Warnings (no-unused-vars и т.п.) — допустимы, не блокируют деплой.

### Шаг 2 — Ручная проверка изменённых API-файлов
Перед коммитом проверь каждый изменённый файл в `api/`:
- [ ] Все template literals закрыты (открывающий `` ` `` имеет парный закрывающий)
- [ ] Нет незакрытых скобок `{`, `(`, `[`
- [ ] Функция `export default async function handler` присутствует и закрыта

### Шаг 3 — Проверь лимит Serverless Functions
```bash
ls api/*.js | Where-Object { $_.Name -notmatch "^_" } | Measure-Object
```
Vercel Hobby план: **максимум 12 функций** (файлы без префикса `_`).
Если больше 12 — деплой упадёт. Переименуй вспомогательные файлы в `_filename.js`.

## ⚠️ КРИТИЧЕСКИЕ ПРАВИЛА (нельзя нарушать)

### Безопасность
- **НИКОГДА** не хардкодить API-ключи, токены, секреты в коде. Только `process.env.VAR_NAME`.
- Firebase Admin SDK ключи — только через env переменные.

### Архитектура UID (критично для подписок)
- Все пользователи идентифицируются через **`tg_{telegramId}`** — это стабильный UID.
- Никогда не создавай новых систем авторизации без понимания этой архитектуры.
- Путь к подписке в Firestore: `users/tg_{telegramId}/subscription/current`

### Платёжная система (ЮKassa)
- Webhook endpoint: `api/payment-webhook-yookassa.js`
- IP-адреса ЮKassa в whitelist: `185.71.76.*`, `185.71.77.*`, `77.75.153.*`, `77.75.154.*`
- **НЕ трогай whitelist IP без явного указания** — это уже обожгло нас дважды.
- Metadata платежа всегда должна содержать `telegramId` для корректного resolve UID.

### Лимиты квоты Gemini API
- Не вызывай Gemini API в циклах без явного `await` между итерациями.
- Не генерируй изображения/текст в одном запросе более 4 раз подряд.
- При ошибке `429` (quota exceeded) — возвращай понятное сообщение пользователю, не ретраить бесконечно.
- **Никогда не вызывай AI API в фоне без явного действия пользователя** — это сжигает квоту незаметно.

### generate-image.js (особое внимание)
- Это самый большой и сложный файл проекта (~2900 строк). 
- **Любое редактирование** этого файла требует проверки: `npm run lint` после сохранения.
- Файл содержит большие multiline template literals с промптами — легко случайно сломать.
- После правки обязательно проверь: `Invoke-WebRequest -Method OPTIONS -Uri "https://seller-studio-ai.ru/api/generate-image"` должен вернуть 200.

## Правила разработки

1. **Фоновые задачи**: Вся тяжелая логика вынесена в Serverless Functions (`api/`). Не делай долгих блокирующих вызовов.
2. **Удаление фона**: `@imgly/background-removal` используется на стороне клиента для снижения нагрузки на сервер.
3. **Тестирование после деплоя**: После каждого `vercel --prod` — проверь OPTIONS-запрос к `/api/generate-image`. 200 = модуль загрузился. 500 = синтаксическая ошибка в файле.

## Известные проблемы и их решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| `500 SyntaxError` на `/api/generate-image` | Незакрытый template literal в файле | `npm run lint`, найди строку с ошибкой, исправь |
| Подписка не активируется после оплаты | Webhook ЮKassa отклонён (неверный IP) | Проверь whitelist в `payment-webhook-yookassa.js` |
| `Function count limit exceeded` при деплое | Более 12 файлов без `_` префикса в `api/` | Переименуй вспомогательные файлы в `_name.js` |
| Пользователь теряет подписку | Анонимный UID вместо `tg_{id}` | Проверь что auth использует Custom Token |
